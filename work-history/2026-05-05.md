## 세션 2 — 미해결 이슈 처리

### 이슈 1: POST /api/auth/sign-up Swagger 201 응답 스키마 미표시 → 수정 완료

**원인**: SpringDoc은 `@ResponseStatus(HttpStatus.CREATED)` 사용 시 201 응답 스키마를 자동 추론하지 않음.

**수정**: `AuthController.kt`에 `@ApiResponse(responseCode = "201", content = [Content(schema = Schema(implementation = TokenResponse::class))])` 명시

```kotlin
@Operation(
    summary = "회원가입",
    responses = [ApiResponse(
        responseCode = "201",
        content = [Content(schema = Schema(implementation = TokenResponse::class))],
    )],
)
```

### 이슈 2: 운영 DB schedules.date 레거시 데이터 → 해당 없음

**결론**: 프로젝트가 아직 미배포 상태이므로 운영 DB에 실제 데이터가 없음. 이슈 자체가 해소됨.

V7 마이그레이션(`UPDATE schedules s SET date = (SELECT t.date FROM todos t WHERE t.id = s.todo_id)`)이 이미 올바른 백필 로직을 포함하고 있으므로, 첫 배포 시에도 문제 없음.
