## [추가] CORS 설정

### 컨텍스트
`POST /api/auth/complete-profile` 호출 시 브라우저에서 `strict-origin-when-cross-origin` CORS 오류 발생. CORS 설정이 전혀 없었음.

### 작업 내역
- `SecurityConfig.kt`: `.cors { it.configurationSource(corsConfigurationSource()) }` 추가, `corsConfigurationSource()` 빈 추가
- `application-local.yml`: `cors.allowed-origins: http://localhost:3000` 추가
- `application-dev.yml`: `cors.allowed-origins: ${CORS_ALLOWED_ORIGINS}` 추가
- `application-prod.yml`: `cors.allowed-origins: ${CORS_ALLOWED_ORIGINS}` 추가

### CORS 설정 내용
```
allowedMethods: GET, POST, PUT, DELETE, OPTIONS
allowedHeaders: *
allowCredentials: true
maxAge: 3600
```

### 환경변수 (dev/prod 배포 시 필수)
```
CORS_ALLOWED_ORIGINS=https://your-frontend-domain.com
```
> 여러 origin은 쉼표로 구분: `https://a.com,https://b.com`

## [추가] 구글 로그인 후 API 호출 시 password null 오류 수정

### 원인
`CustomUserDetailsService.loadUserByUsername`에서 `User.withUsername(...).password(user.password)`를 호출할 때, 구글 OAuth 사용자는 `password = null`이므로 Spring Security가 `IllegalArgumentException: password cannot be null` 발생.

### 수정
- `CustomUserDetailsService.kt`: `user.password` → `user.password ?: ""`
