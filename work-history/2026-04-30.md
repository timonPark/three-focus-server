## 아키텍처 주요 결정사항

### Repository 패턴
- **JpaRepository**: 단순 CRUD (save, deleteById 등)
- **jOOQ QueryRepository**: 복잡한 조회 쿼리 (findBy 계열, JOIN 쿼리)
- jOOQ는 DSL.table(), DSL.field() 방식 사용 (코드 생성 없이 plain string)

### ScheduleQueryRepository.findDailyScheduleByUserIdAndDate
- top3 + todos + schedules 를 단일 JOIN 쿼리로 조회
- LEFT JOIN schedules (시간 미배치인 Top3 항목도 포함)
- DailyScheduleItemResponse를 repository에서 직접 projection

### 소유권 검증
- 모든 변경 API에서 `todo.userId != userId` 체크 → FORBIDDEN
- TodoService에 `findOwned(userId, todoId)` private 헬퍼 존재

### Todo 삭제 cascade
- TodoService.delete() 순서: scheduleRepository.deleteByTodoId → top3Repository.deleteByTodoId → todoRepository.deleteById
- DB 레벨에도 ON DELETE CASCADE 설정되어 있음 (V1__init.sql)

### Security
- JWT stateless 방식
- permitAll: /api/auth/**, /swagger-ui/**, /v3/api-docs/**, GET /api/shares/*
- 나머지 모두 인증 필요
- **미해결**: SecurityConfig에 AuthenticationEntryPoint 미설정으로 인해 미인증 요청 시 401이 아닌 403 반환됨

### 현재 실패 테스트 (5개) — 모두 동일 원인
```

     → Spring Security 6 기본값이 Http403ForbiddenEntryPoint

수정 방법: SecurityConfig에 아래 추가
.exceptionHandling {
    it.authenticationEntryPoint { _, response, _ ->
        response.sendError(HttpServletResponse.SC_UNAUTHORIZED)
    }
}
```
